└─────────────────────────────────────────────────────────────────────┘

TITRE :
  RBAC — Gestion des Accès par Rôle — Moindre Privilège

4 RÔLES :
  admin         → Accès complet : monitoring · users · toutes données · configuration · exports
  sous-admin    → Gestion opérationnelle : étudiants · classes · paiements · factures · rapports
  comptable     → Finance uniquement : lecture paiements · génération factures · exports
  étudiant/parent → Portail personnel : propres données uniquement · lecture seule · pas d'admin

IMPLÉMENTATION :
  checkRole(['admin','sous-admin']) sur chaque route
  Re-lecture rôle Firestore à CHAQUE requête (jamais depuis JWT uniquement)
  Si rôle révoqué → bloqué immédiatement à la prochaine requête

CE QUE VOUS DITES :
  "4 rôles : admin, sous-admin, comptable et étudiant/parent. Le rôle est
  relu dans Firestore à chaque requête — jamais depuis le JWT uniquement.
  Si un rôle est révoqué, l'accès est bloqué immédiatement."

2.1 Agent macOS non visible (Disconnected)
  Cause    : Mauvaise adresse manager dans /Library/Ossec/etc/ossec.conf
  Solution : Reconfiguration ossec.conf + sudo /Library/Ossec/bin/wazuh-control restart
  Résultat : Agent 001 Active ✓

2.2 FIM sans résultats (43200s par défaut)
  Cause    : Fréquence syscheck = 43 200 sec (12h) — aucun scan pendant démo
  Solution : <frequency>43200</frequency> → <frequency>60</frequency>
  Action + : sudo bash -c "echo '# wazuh-test' >> /private/etc/hosts" pour forcer FIM
  Résultat : 436 420+ alertes FIM ✓

2.3 NVD Feed CVE non téléchargé (payant)
  Cause    : Wazuh 4.7+ requiert abonnement pour feed NVD complet (HTTP 403)
  Solution : Utilisation syscollector Wazuh natif pour agent Docker (Linux)
  Résultat : 17 CVE détectées via packages installés ✓

2.4 Backend Node.js ne démarre pas (bcrypt DLL Windows)
  Cause    : Module bcrypt compilé sous Windows (PE32+) — incompatible macOS (Mach-O)
  Solution : rm -rf node_modules/bcrypt && npm install bcrypt && npm rebuild bcrypt
  Résultat : Backend opérationnel sur http://127.0.0.1:5001 ✓

2.5 Firebase Emulator timeout (defineSecret)
  Cause    : defineSecret() appelle APIs Firebase réseau indisponibles en mode offline
  Solution : Création server_local.js avec mock des modules firebase-functions
  Résultat : Express app sur port 5001 — logs YNOV-APP générés ✓

2.6 Règles Wazuh — champ 'action' réservé
  Cause    : <field name='action'> est un champ statique réservé dans wazuh-analysisd
  Solution : <field name='action'>AUTH_FAILURE</field> → <match>action=AUTH_FAILURE</match>
  Résultat : 7 règles YNOV-APP actives — Level 3 à 14 ✓

2.7 Permissions macOS SIP — /etc write denied
  Cause    : System Integrity Protection empêche tout accès en écriture à /etc
  Solution : Utiliser /private/etc/hosts : echo '# test' >> /private/etc/hosts
  Résultat : Alerte FIM /private/etc/hosts modified — Rule 550 Level 7 ✓

2.8 PPTX corrompue (Bad CRC-32)
  Cause    : Fichier PPTX corrompu suite à modification python-pptx antérieure
  Solution : Reconstruction ZIP — lecture fichier/fichier — remplacement image corrompue
  Résultat : PPTX réparé — 102 fichiers valides ✓
